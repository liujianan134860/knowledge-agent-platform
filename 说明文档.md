# Knowledge Agent Platform — 部署说明文档

## 一、本地启动

### 1. 环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | 推荐 Eclipse Temurin 17 |
| Maven | 3.6+ | 项目自带 Maven Wrapper（`mvnw`），无需全局安装 |
| API Key | — | DeepSeek + 阿里云 DashScope（见下方） |

### 2. API Key 准备

需要申请两个 API Key（均为付费，按量计费）：

- **DeepSeek API Key**：https://platform.deepseek.com → API Keys → 创建
- **阿里云 DashScope API Key**：https://dashscope.console.aliyun.com → API-KEY 管理 → 创建（用于文本向量化 Embedding + 评测模型）

### 3. 配置文件

```bash
# 1. 复制环境变量模板
cp .env.example .env

# 2. 编辑 .env，填入你的 API Key
#    DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
#    QWEN_API_KEY=sk-xxxxxxxxxxxxxxxx
#    JWT_SECRET=任意随机字符串（生产环境务必修改）
```

其他变量保持默认即可。完整变量说明见 `.env.example` 注释。

### 4. 启动服务

```bash
# 开发模式启动（支持热加载）
mvn spring-boot:run

# 或者先打包再运行
mvn clean package -DskipTests
java -jar target/*.jar
```

启动日志中出现 `Started KnowledgeAgentPlatform in X seconds` 即表示成功。

### 5. 访问地址

| 地址 | 说明 |
|------|------|
| http://localhost:8081 | 前端界面（Agent Console / 知识库 / 上传 / 追踪） |
| http://localhost:8081/swagger-ui/index.html | Swagger API 文档 |
| http://localhost:8081/h2-console | H2 数据库控制台（开发调试） |
| http://localhost:8081/actuator/health | 健康检查 |

### 6. 注册用户 & 登录

前端界面直接注册，或通过 API：

```bash
# 注册
curl -X POST http://localhost:8081/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"test1234"}'

# 登录（返回 JWT token）
curl -X POST http://localhost:8081/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"test1234"}'
```

## 二、上线云服务器

提供 **两种方式**：一键部署（Render）和 手动部署（任意 VPS）。

---

### 方式 A：Render 一键部署（最简单）

项目已包含 `Dockerfile` + `render.yaml`，推送代码后自动拉起。

**步骤：**

1. 将项目推送到 GitHub 仓库
2. 打开 [Render Dashboard](https://dashboard.render.com) → **Blueprints** → **New Blueprint Instance**
3. 连接你的 GitHub 仓库，Render 自动读取 `render.yaml`
4. 在环境变量页面填入：
   - `DEEPSEEK_API_KEY` — DeepSeek API Key
   - `QWEN_API_KEY` — 阿里云 DashScope API Key
   - `GITHUB_MCP_TOKEN` — GitHub Token（可选，不启用 MCP 工具可不填）
5. 点击 **Apply**，等待约 10 分钟部署完成
6. Render 会分配一个免费域名：`https://knowledge-agent-platform.onrender.com`

**Render Free 计划限制：**
- 15 分钟无请求自动休眠，下次唤醒约 30-60 秒
- 每月 750 小时免费额度（刚好够单实例运行）
- 数据存储在容器内，重新部署会丢失（建议定期备份 `data/` 目录）

---

### 方式 B：手动部署 VPS（阿里云 / 腾讯云 / AWS 等）

以 **Ubuntu 22.04** 为例，其他 Linux 发行版同理。

#### Step 1：服务器环境准备

```bash
# 安装 JDK 17
sudo apt update
sudo apt install -y openjdk-17-jre-headless

# 验证
java -version
```

#### Step 2：打包上传

```bash
# 在本地项目根目录执行
mvn clean package -DskipTests

# 上传到服务器（替换为你的服务器 IP）
scp target/knowledge-agent-platform-0.0.1-SNAPSHOT.jar user@your-server:/opt/knowledge-agent/app.jar
scp .env user@your-server:/opt/knowledge-agent/.env
```

首次部署还需创建数据目录：

```bash
# 在服务器上
mkdir -p /opt/knowledge-agent/data
```

#### Step 3：配置环境变量

编辑服务器上的环境变量：

```bash
# 在服务器上
vim /opt/knowledge-agent/.env
```

务必修改的值：
- `DEEPSEEK_API_KEY` — 填入真实 Key
- `QWEN_API_KEY` — 填入真实 Key
- `JWT_SECRET` — 用 `openssl rand -hex 32` 生成随机密钥
- `PORT` — 保持 8081 或自定义

#### Step 4：注册为系统服务（systemd）

```bash
sudo tee /etc/systemd/system/knowledge-agent.service << 'EOF'
[Unit]
Description=Knowledge Agent Platform
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/knowledge-agent
EnvironmentFile=/opt/knowledge-agent/.env
ExecStart=/usr/bin/java \
  -Xms256m -Xmx1024m \
  -Djava.security.egd=file:/dev/./urandom \
  -jar /opt/knowledge-agent/app.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF
```

```bash
# 启动并设置开机自启
sudo systemctl daemon-reload
sudo systemctl enable knowledge-agent
sudo systemctl start knowledge-agent

# 查看状态
sudo systemctl status knowledge-agent

# 查看日志
sudo journalctl -u knowledge-agent -f
```

#### Step 5：Nginx 反向代理（可选，推荐）

安装 Nginx：

```bash
sudo apt install -y nginx
```

配置反向代理：

```bash
sudo tee /etc/nginx/sites-available/knowledge-agent << 'EOF'
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名

    client_max_body_size 20M;

    location / {
        proxy_pass http://127.0.0.1:8081;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 长连接支持
        proxy_buffering off;
        proxy_read_timeout 300s;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/knowledge-agent /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

#### Step 6：配置 HTTPS（强烈建议）

```bash
# 安装 certbot
sudo apt install -y certbot python3-certbot-nginx

# 自动配置 SSL 证书
sudo certbot --nginx -d your-domain.com

# 证书自动续期
sudo certbot renew --dry-run
```

#### Step 7：防火墙

```bash
# 仅开放 80 和 443（Nginx），不直接暴露 8081
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp   # SSH
sudo ufw enable
```

## 四、数据库说明

| 环境 | 数据库 | 位置 |
|------|--------|------|
| 开发 | H2（内嵌文件数据库） | `data/knowledge-agent-platform.mv.db` |
| 生产（未配置 PostgreSQL） | H2（同上） | 容器/服务器 `data/` 目录 |
| 生产（已配置 PostgreSQL） | PostgreSQL + pgvector | 需外部 PostgreSQL 实例 |

切换到 PostgreSQL 只需在 `.env` 中取消注释并填入连接信息：

```bash
POSTGRES_URL=jdbc:postgresql://your-pg-host:5432/knowledge_agent
POSTGRES_DRIVER=org.postgresql.Driver
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your-password
HIBERNATE_DIALECT=org.hibernate.dialect.PostgreSQLDialect
PGVECTOR_INIT_SCHEMA=true
```

向量存储也会自动从 SimpleVectorStore（内存 + 文件）切换到 PgVectorStore（PostgreSQL + pgvector）。

## 六、常见问题

**Q: 启动报 `JWT_SECRET` 相关错误？**
A: `.env` 中的 `JWT_SECRET` 不能为空，生成随机串：`openssl rand -hex 32`

**Q: deepseek-v4-pro 工具调用报 400？**
A: v4-pro 的 thinking mode 在多轮工具调用时有兼容问题，将 `DEEPSEEK_MODEL` 改为 `deepseek-chat` 即可。

**Q: 前端乱码？**
A: 确保 Nginx 配置了 `charset utf-8;`，且 `.env` 文件以 UTF-8 编码保存。

**Q: Render 休眠后数据丢失？**
A: Render Free 计划的容器文件系统在休眠/重新部署时会重置。建议使用 PostgreSQL 外部数据库（Render 提供免费的 PostgreSQL 实例）存储持久数据。
