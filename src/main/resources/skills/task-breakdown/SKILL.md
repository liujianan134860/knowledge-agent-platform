# Task Breakdown Skill

## 适用场景
用户有一个大目标或项目，需要将其拆解成具体可执行的小任务。

## 输入信息
- 用户的大目标描述
- 可选的截止日期
- 可选的优先级偏好

## 执行步骤
1. 理解用户目标的完整范围
2. 分析目标的关键组成部分
3. 将目标分解为 3-8 个具体子任务
4. 确定每个子任务的优先级
5. 如有关键依赖，标注依赖关系
6. 逐个子任务使用 task.create 创建
7. 输出任务树和下一步行动建议

## 输出格式
```
## 任务拆解 — [目标名称]

### 子任务列表
1. [任务名] (HIGH) — 建议完成时间: YYYY-MM-DD
2. [任务名] (HIGH)
3. [任务名] (MEDIUM)
...

### 依赖关系
- 任务2 依赖 任务1

### 🎯 下一步
先开始 [任务1]
```

## 注意事项
- 子任务控制在 3-8 个
- 避免过度拆解导致管理负担
- 优先拆解最关键的部分
